Iterative
Review Cycle

THE OBJECTIVE

Review, refine, repeat. Verbal feedback guides the agent through multiple polish cycles.

REVIEW PRIORITY

1. Functionality first
2. User experience
3. Visual polish
4. Edge cases

Multiple iterations are normal

THE REVIEW LOOP

Test

Run it, click through it, break it

→

Speak

Verbal feedback on what needs fixing

→

Iterate

Agent implements, you test again

EXAMPLE VOICE FEEDBACK

"The login form works but the error messages are too harsh. Soften the language to match our narrative guidelines. Also, the submit button needs more padding — it feels cramped."

BE SPECIFIC

"The button is too small" → "Add 8px padding to the submit button"

REFERENCE DOCS

"Match the design system" → Agent checks design-system.md

KNOW WHEN TO STOP

Diminishing returns are real. Ship when it's good enough. Perfect is the enemy of deployed.
