PARALLEL WORK

Sub-agents

Claude can spawn child agents to work on subtasks in parallel, then combine results.

How It Works

Main agent delegates complex tasks to specialized sub-agents. Each sub-agent handles a portion, returning results to the main agent.

Example Use Cases

"Refactor all API endpoints" - spawns agents per file
"Add tests for each module" - parallel test generation
"Research and implement" - one researches, one codes

Voice Command Example

"Refactor all the API handlers to use async/await and add error handling to each one"

Benefit: Complex tasks complete faster with parallel execution
