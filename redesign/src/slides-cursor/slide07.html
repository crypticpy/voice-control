WHAT RULES DO

Define how the AI behaves for your project — coding standards, patterns, constraints.

TWO APPROACHES

Legacy: .cursorrules file
Modern: .cursor/rules/ folder

Commit rules to git for team sharing

RULES CONFIGURATION

LEGACY (STILL WORKS)

.cursorrules

Single markdown file in project root. Simple but limited.

RECOMMENDED

.cursor/rules/

Folder with .mdc files. Granular control per component/feature.

EXAMPLE .cursor/rules/ STRUCTURE

.cursor/rules/
  general.mdc       # Project-wide standards
  react.mdc         # React component patterns
  api.mdc           # API design rules
  security.mdc      # Security constraints

GLOBAL RULES

Cursor Settings → General → Rules for AI — applies to all projects

BEST PRACTICE

Keep each .mdc file under 500 lines for optimal performance
