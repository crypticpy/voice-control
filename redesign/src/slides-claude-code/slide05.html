KEY WORKFLOW

Planning
Mode

THE CONCEPT

Think before you code. Plan mode lets you design your approach before making changes.

HOW TO ENTER

/plan command
Shift+Tab twice
Start prompt with "plan"

Always plan before PRs

WHY PLANNING MODE MATTERS

Read

Explore codebase
without changes

Think

Design approach
before committing

Iterate

Refine plan with
feedback

EXAMPLE WORKFLOW

# Enter planning mode
/plan

"I need to add user authentication.
Review the codebase and propose an approach."

# Claude explores, then presents plan
# You approve or refine
# Exit plan mode to implement

CONTEXT BENEFIT

Uses sub-agents for exploration — keeps main context clean

BEST PRACTICE

For PRs, always plan first — iterate on approach before writing code
