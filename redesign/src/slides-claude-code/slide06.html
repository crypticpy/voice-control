PARALLEL WORK

Sub-Agents

WHAT THEY ARE

Specialized agents that work in parallel, each with their own context window.

WHY USE THEM

Keep main context clean
Parallel task execution
Specialized capabilities
Different models per task

Context management is key

BUILT-IN SUB-AGENTS

Explore

Fast codebase scanning. Uses Haiku for speed. Returns findings to main thread.

Plan

Architecture planning. Reads full context. Designs implementation approach.

Bash

Command execution. Runs shell commands. Reports results back.
