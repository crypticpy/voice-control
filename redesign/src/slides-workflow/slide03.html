THE PRINCIPLE

Everything lives in Git. Always. This is non-negotiable for voice-first development.

WHY FIRST?

Agent commits frequently
Creates natural checkpoints
Enables rollback on mistakes
Documents project evolution

Session 2: We'll cover repo setup templates and best practices in detail.

VOICE COMMANDS TO START

"Create a new repo for this project"

gh repo create project-name --public --clone

"Initialize with a basic structure"

Creates README, .gitignore, src/, docs/

COMMIT FREQUENCY

Tell your agent: "Commit after each significant change." Small, frequent commits beat monolithic ones.

BRANCH STRATEGY

Start simple: work on main for solo projects. Add branches when collaborating or experimenting.

BEST PRACTICE

Have the agent commit after: new files created, features completed, bugs fixed, documentation added. Each commit is a restore point.

NEW TO GIT?

Don't worry if you don't have templates set up yet. We'll cover repo structure, CLAUDE.md files, and agent-friendly project layouts in Session 2.
